Interface changes
===================

-> Interface means collection of abstract methods

Note: The method which doesn't contain body is called as abstract method.

-> When a class is implementing interface its mandatory, that class should implement all abstract methods of that interface othewise class can't be compiled.

```
interface Vehicle {
	public abstract void startVechicle ( );

	private void m1(){
		// logic
	}
}

class Car implements Vehicle {

	public void startVehicle ( ) {
		// logic to start car
	}
}

class Bus implements Vehicle {
	public void startVehicle ( ) {
		// logic to start  bus
	}
}


class Bike implements Vehicle {
	public void startVehicle ( ) {
		// logic to start  bike
	}
}
```


=> If we add new method in interface then Car, Bike and Bus will fail at compile time.

=> To overcome above problem we will use Default & Static methods


** Interface can have concreate methods from 1.8v

** Interface concrete method should be default or static


```
public interface Vehicle {
	
	public void start();

	public default void stop(){
		// logic
		m1();
	}

	public static void clean(){
		// logic
		m1();
	}

	private void m1(){
		// logic
	}
}
```

Ex:  forEach ( ) method added in java.util.Iterable interface as default method in 1.8v


----------------------------------------------------------------------

A ---- default m1()

B ---- default m1()

When two interfaces have the same default method, the implementation class must override that method. Otherwise Java will get confused: should it call A.m1() or B.m1()?


class C implements A, B {
	
	void m1(){
		// specific logic
	}
}

** interface default methods we can override in impl classes

** interface static methods we can't overide in impl classes

** We can write multiple default & static methods in interface

** Default & Static methods introduced to provide backward compatability

** When two interfaces have the same default method, the implementation class must override that method.

** In interface we can write private concrete methods also to use only inside that interface (It is from Java 9 onwards)

** Private methods we can't call from implementation classes

** private methods we can't override

** Private methods in interfaces were introduced in Java 9. They are used to write reusable helper logic inside the interface and avoid duplicate code in default and static methods.

import java.util.Arrays;
import java.util.List;
import java.util.function.Predicate;


public class Main {
    public static void main(String[] args) {

        Predicate<Integer> p = i -> i > 10;
        System.out.println(p.test(5));
        System.out.println(p.test(15));

        System.out.println("===========================");

        String[] names = {"Anushka", "Anupama", "Deepika", "Kajol", "Sunny"};

        Predicate<String> p1 = name -> name.charAt(0) == 'A';

        for (String name : names) {
            if (p1.test(name)) {
                System.out.println(name);
            }
        }

        System.out.println("===========================");


        Employee e1 = new Employee("Anil", "Chennai", "DevOps");
        Employee e2 = new Employee("Rani", "Pune", "Networking");
        Employee e3 = new Employee("Ashok", "Hyd", "DevOps");
        Employee e4 = new Employee("Ganesh", "Hyd", "DevOps");

        List<Employee> empsList = Arrays.asList(e1, e2, e3, e4);

        Predicate<Employee> localtionPredicate = e -> e.location.equals("Hyd");
        Predicate<Employee> deptPredicate = e -> e.dept.equals("DevOps");

        // Predicate Joining
        Predicate<Employee> empPredicate = localtionPredicate.and(deptPredicate);

        for (Employee emp : empsList) {
            if (empPredicate.test(emp)) {
                System.out.println(emp.name);
            }
        }


    }
}

Supplier Functional Interface
===============================

-> Supplier is a predefined functional interface introduced in java 1.8v

-> It contains only one abstract method that is get ( ) method

-> Supplier interface will not take any input, it will only returns the value.

import java.util.Random;
import java.util.function.Supplier;

public class SupplierDemo {

    public static void main(String[] args) {

        Supplier<String> s = () -> {
            String otp = "";
            for (int i = 1; i <= 6; i++) {
                otp = otp + (int) (Math.random() * 10);
            }
            return otp;
        };

        System.out.println(s.get());
        System.out.println(s.get());
        System.out.println(s.get());
        System.out.println(s.get());

        System.out.println("=======================");

        int otpLength = 8;
        String characters = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789";
        Random random = new Random();

        Supplier<String> otpSupplier = () -> {
        StringBuilder otp = new StringBuilder();
        for (int i = 0; i < otpLength; i++) {
            int index = random.nextInt(characters.length());
            otp.append(characters.charAt(index));
        }
            return otp.toString();
        };

        System.out.println("Generated OTP: " + otpSupplier.get());
        System.out.println("Generated OTP: " + otpSupplier.get());
        System.out.println("Generated OTP: " + otpSupplier.get());
        System.out.println("Generated OTP: " + otpSupplier.get());

    }
}

===========
Stream API
===========

-> Stream API introduced in java 1.8v

-> Stream API is used to process the data

-> It is used to process collections of data in a clean and functional programming style.

Note: Collections are used to store the data

-> Stream API is one of the major features added in java 1.8v

-> Stream in java can be defined as sequence of elements that comes from a source.

-> Source of data for the Stream can be array or collection


====================================
Few Important Points About Streams
====================================

1) Stream is not a data structure. Stream means bunch of operations applied on source data. Source can be collection or array.

2) Stream will not change original data structure of the source (It will just process the data given by the source.)


#### Main Advantages with Stream API 

 ** less code
 ** Clean code
 ** Readable Code
 ** Functional Style Code
 ** Easy Data Processing Logic

===============
Stream Creation
===============

-> In Java we can create Stream in 2 ways

			1) Stream.of(e1, e2, e3, e4)

			2) stream () method


===================
Stream Operations
===================			

-> Stream API provided several methods to perform Operations on the data.

-> We can divide Stream api methods into 2 types

		1) Intermediate Operational Methods

		2) Terminal Operational Methods

-> Intermediate Operational methods will perform operations on the stream and returns a new Stream

		ex: filter (), map (), sorted(), distinct(), limit(), skip() etc...		

-> Terminal Operational methods will take input and will provide result as output.

		Ex:  count(), collect (), forEach(), reduce(), min(), max(), anyMatch().. 


==================
Stream API Flow
==================


Collection Data ---> Stream ---> Intermediate Operation ----> Terminal Operation


===================
Stream API Methods
===================

1) filter (Predicate p) : It is used to filter the data 

2) map (Function f) : Used to transform the data from one format to another format.

3) forEach(Consumer c) : It is used to iterate the data

4) collect ( ) : It is used to collect the stream result into List, Set, Map etc...

5) count ( ) : used to count no.of records

6) sorted () : Used to perform natural sorting

7) distinct () : Used to remove duplicate elements

8) limit ( ) : To limit records

9) skip ( ) : To skip records

10) anyMatch ( ) : Checks whether at least one record matches condition.

11) allMatch ( ) : Checks whether all records match condition.

12) noneMatch ( ) : Checks whether no record matches condition.

13) findFirst ( ) : Returns first matching record.

14) findAny ( ) : Returns any matching record.

15) min ( ) : Find minimum number

16) max ( ) :  Find maximum number

17) reduce ( ) : Used to reduce multiple values into a single value.

Why Sealed Classes?
=======================

=> Sealed classes are used to control inheritance.

=> They are useful when we know all possible child types.


1) Payment

        -- CreditCardPayment
        -- DebitCardPayment
        -- UpiPayment


2) User

        -- AdminUser
        -- CustomerUser
        -- VendorUser


3) Vehicle

        -- Car
        -- Bus
        -- Truck


## syntax ::

public sealed class Vehicle permits Car, Bus, Truck {

}

public sealed class Payment permits CreditCardPayment, UpiPayment, NetBankingPayment {

}


Note : Child classes must be one of these

        1) final (class can't be extended)

        2) sealed (class can be extended only by permitted classes)

        3) non-sealed (Removes sealing restriction and allows normal inheritence)

====================
Sealed Interface
====================

=> sealed interface is used to control who can implement our interface.

public sealed interface PaymentRequest permits UpiRequest, CardRequest, NetBankingRequest {
}

===========================================
Important Rules of Sealed Classes
============================================

1) Sealed class must declare permitted sub classes

2) Permitted subclass must directly extend sealed class

3) Child class must be final, sealed or non-sealed

4) sealed classs can be abstract

5) Sealed interfaces also allowed.
